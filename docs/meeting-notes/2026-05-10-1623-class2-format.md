## Discussion

### Item 1 — Pattern characterisation

**Archie:** Two distinct shapes coexist in zkm's `docs/meeting-notes/`:

- **True persona-meetings** (`2026-05-07-object-storage.md`, `2026-05-08-next-plugins.md`, `2026-05-08-2254-privacy-audit.md`): full live dialogue — named speakers, argued convergence.
- **Protocol overviews** (`2026-05-08-2101-encoding-audit.md`, `2026-05-10-1628-n9a-value-normalization.md`): Discussion section is a structured summary — "TODO classification", "Design decisions", "Implementation findings", "Test results". No persona names in Discussion.

The encoding-audit note states it explicitly: *"Dispatched to EnterPlanMode / explore → design → implement flow (no persona scaffolding for Class 2)."* — the skill behaving as designed, but the artifact filename, location, and template-shape are meeting-note ones.

**Sage:** This is the no-arg dispatch. Class 1 → implement (no artifact). Class 2 → EnterPlanMode + native explore-design (no personas) → writes a meeting note. Class 3 → full persona meeting. The Class 2 artifact location isn't pinned in the spec; convention crystallised as `docs/meeting-notes/` because the index lives there.

**Riku:** "Abuse" framing assumes personas-only format. The spec was *amended* to define a Class 2 dispatch path. The spec is working as written. The question is whether the spec is wrong, not zkm.

### Item 2 — Costs of mixing

**Riku:** Three potential costs:
1. **Reader confusion** — index expects persona dialogue; finds a summary. Mild.
2. **Search/grep noise** — future meetings citing a Class 2 note as "what did personas decide?" find no persona positions. Real.
3. **Convention drift** — format becomes whatever Claude writes for Class 2; gradual erosion. Unverified.

**Archie:** Counter-cost of separating: split audit trail; `docs/meeting-notes/` loses single-source-of-truth function.

**Sage:** Cheapest fix: header tag. The skill knows the class at dispatch time.

**Riku (pre-empt, empirical-pilot):** "You might prefer to observe before intervening — count how often a Class 2 note is cited as if it had persona decisions." CLAUDE.md "observe before preventing" backs it.

### Item 3 — Initial options

- **A.** Do nothing.
- **B.** Header tag (`**Type:** Class 2 planning session`).
- **C.** Separate directory (`docs/plans/`). Migration cost.
- **D.** Different `## Section` headings for Class 2 (no `## Discussion`).
- **(Riku's radical):** Class 2 produces no artifact.

**Petra:** N=2 fails for C — no tooling consumer benefits from directory split. **Archie:** Radical fails — TODO.md + meeting-style.md cross-reference these for closure; removing the artifact orphans citations.

### Item 4 — Tobias's driver

**Tobias:** Format dilution + misleading citations (functional). "Wrong artifact entirely" is partial — *OK to store Class 2 as a standalone note IF framed clearly as "no meeting happened, only the following planning."*

**Riku:** Then `**Type:** Class 2` is too quiet. The *absence of a meeting* must be plain at first glance.

**Petra:** Drop `## Discussion` for Class 2; use `## Plan` / `## Implementation findings`. The H2 heading was the dilution surface.

**Sage:** Combined template (B + D + explicit framing): Mode field replaces Attendees; distinct body sections.

### Item 6 — Concerns against β / convergence

**(a) Plan execution:** Move happens after ExitPlanMode (step 4 in the lifecycle). Plan execution unaffected. Pins *when* the move can happen, not *whether*.

**(b) Permission prompts:** Read+Write avoids cross-dir mv entirely. Plan file is auto-cleaned by Claude Code. β reduces to α' — Read plan content, Write polished record at destination.

**Convergence (α'):**
- Class 2 artifact at same path/naming as Class 3.
- Distinct Class 2 template: `**Mode:**` instead of `**Attendees:**`; body uses `## Context / ## Plan / ## Implementation findings / ## Decisions / ## Action items` (no `## Discussion`).
- Content synthesises the plan file; plan file left to auto-cleanup.
- `format.md` documents both templates in one place.
- No allowlist changes, no worktree, no commit prefixes.

## Decisions

1. **Class 2 dispatches produce a planning record at `<root>/docs/meeting-notes/YYYY-MM-DD-HHMM-<slug>.md`** using a distinct **Class 2 template**:
   - `**Mode:** Class 2 planning record (no meeting was held — plan-mode output)` (replaces `**Attendees:**`).
   - Body: `## Context`, `## Plan`, `## Implementation findings`, `## Decisions`, `## Action items`. **No `## Discussion`.**
   - Content is the polished synthesis of the plan file; plan file left to Claude Code's auto-cleanup.
2. **`format.md` documents both templates** — Class 2 and Class 3 — in a new subsection.
3. **Existing Class 2 notes are not migrated.** Grep for `**Mode:** Class 2` to identify new-format ones. Legacy notes stay as historical artifacts.
4. **Out of scope** (with reasons):
   - Worktree/separate-branch: preventive, no observed pain; deferred.
   - Commit-prefix discipline: user declined; natural separation already happens.
   - Class 1 artifact: commits + TODO suffice.
   - Cross-project migration: per-project authors handle lazily.
