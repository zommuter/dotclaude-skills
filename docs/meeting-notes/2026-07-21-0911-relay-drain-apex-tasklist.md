## Decisions

The node `drain-driver.mjs` stays the **headless/cron** substrate. A **stateless `drain-stop` CLI** is extracted (reads the current runId's `round-result` event tail → `CONTINUE`/`DRAINED`/`BLOCKED`); **both loops consume it** (one predicate, two loops — Riku's guard-parity pre-condition). The `id:23ff` live round is built ONCE and shared, not forked per substrate. *Out of scope:* retiring the node driver; the apex-as-observer hybrid (owner-rejected).
- **D2 — Lane-keyed, supervised-only, LLM-only dispatch.** handoff=Opus, review=Opus (fixed); execute=Sonnet for `[ROUTINE]` / Opus for `[HARD — pool]`, difficulty read **mechanically via the id:4da4-anchored parser** (not substring — id:0d58). Execute agent loads the `/relay executor` contract. `[MECHANICAL]`→mechanical-daemon substrate; `[INPUT — *]`→surface-as-blocked; never an LLM Agent (no LLM/non-LLM mixing). The apex-driver drain is **supervised / explicitly-requested only** — never the no-arg unattended default; cron-wiring it = the forbidden auto-spend. *Out of scope:* the lane-taxonomy rename (id:4f02 `[INPUT — meeting|hands|access]` stands — owner-confirmed this session).
- **D3 — Re-derive TaskList each round.** Durable SSOT = JSONL event feed + relay ledgers on disk. The apex re-derives its TaskList from mechanical state (classify-repo.sh + ROADMAP + events, **scoped to the current runId** — the JSONL is append-only across runs) at the top of every round; never carried from ctx. Session death → restart rebuilds identically. Parallel/merge state stays in `drain-integrate.sh`'s `--merged-so-far` file. Fan-out reuses id:5367 (declared-disjoint-or-serial) + id:2062. *Out of scope:* making the TaskList authoritative.
- **D4 — 3-ledger drained (clear-of-actionable).** Work-source enumerates ROADMAP + TODO + inbox (`[<repo>]`-targeted open items, inbox path via `$RELAY_INBOX` injection). "drained" = all three clear **of actionable work** PLUS the effectively-drained check (fully-`[x]` ROADMAP hiding open TODO, SKILL.md:452); an open `[INPUT — *]`/`[HARD — meeting]` item = **blocked**, not un-drained. Folds `routed:bdee`; bdee stays its own broader item (every repo-scoped relay command), drain is its first consumer. *Out of scope:* the full bdee build for the other repo-scoped commands.
- **D5 — c17c anti-spam brief.** The review agent's prompt hard-wires, verbatim: verify against the RED spec; do NOT re-litigate ratified design; do NOT open a REVIEW_ME box for anything already settled in a meeting note; forbidden from writing the owner-accept marker (executor-contract v10). K=2 dry-round stop guards no-progress ping-pong. Note: c17c's mechanical enforcement half is still owed via 2e5c/07dc. *Out of scope:* building that mechanical half here.
- **D6 — Cron-safety enforced, not documented.** The apex-drain entry refuses to start when stdin is not a tty (`! test -t 0`) unless an explicit override flag (e.g. `--i-know-this-auto-spends-opus`) is passed. Blocks accidental headless/cron invocation of the Opus-spending loop; the override keeps a deliberate scripted supervised run possible. *Out of scope:* any quota-based auto-throttle (the existing quota-stop.sh gate covers spend).
- **D7 — Next-action precedence: review → execute → handoff.** When multiple stages are ready in a round, the apex clears **verification first** (verify landed executor work), then **execute** (ready RED specs / `[ROUTINE]`), then **handoff** (untagged TODO → specs). Bounds work-in-progress — never generates new work while unverified work piles up. *Out of scope:* per-unit priority ordering within a stage (FIFO by ledger order for now).
- **Lease (contract requirement, not a fork).** The apex driver acquires the `claim.sh` relay lease before round 1 and releases on every exit path (mode-blind → correctly excludes a concurrent pool). Standard `/relay` contract, recorded for completeness.

Recorded as an informed override, not a silent choice.

**Decisions (amendment):**
- **A1 — Hard-deny on add.** The pre-commit hook exits nonzero and blocks the commit when a staged *added* line introduces an old-vocab lane tag (`[HARD — pool|meeting|hands|decision gate]`), naming the new-vocab replacement; existing old-vocab tags only WARN (grandfathered). Added-lines-only via `git diff --cached`. *Out of scope:* a report-only mode (rejected in favour of hard-deny + `--no-verify`).
- **A2 — Tag-vs-prose via the anchored parser.** Classification MUST reuse the `id:4da4`-anchored parser / `roadmap-lint.sh` tag detection — never a fresh substring grep (the `id:0d58` false-positive class). The new work is the added-lines-only filter, not re-detecting tags.
