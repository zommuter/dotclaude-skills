## Context

`id:fa1d` ("ONE meeting: who may write the shared thing, and when") was amended 2026-07-19 (mtg-1726) to **un-couple `id:dc5b`** (execute+review ledger collision — ratified separately as C2) from `id:d03d`. Grounding facts carried into the room:
- Mechanical daemon tier (`relay-ro`/`relay-svc`) already provisioned; **LLM executor tier is unscoped** (runs as `tobias`, full write + push credential).
- `id:5937` deny-probe: `permissionMode: auto` denies **nothing** (write-outside-repo, `git push`, network egress all approved); **only OS DAC contained**; `--disallowedTools` is evadable (`id:453a`).
- Motivating instance is **n=1**: an agent self-reported editing a source file in the main checkout instead of its worktree — self-caught, blast radius ~zero, **agent error not adversarial intent**. Honest open question: how often does it go *uncaught*.
- git worktrees **share `.git`**, so file-DAC on the working tree can't stop `git branch -f main`/ref writes; real-remotes dissolves this structurally.
- Mechanism splits into two separable claims: (i) real-remotes topology needs **no** btrfs (`git clone --local` gives an independent `.git`; repos ~20 MB); (ii) **btrfs reflink** as the *copy mechanism* earns its place on independence-without-hardlink-aliasing + **warm** working trees (`node_modules`/`.venv` at O(metadata)).

## Discussion

**Framing (item 1).** Archie: the real-remotes topology stands on architectural merit alone — it dissolves the shared-`.git` write hazard, dissolves orphan-worktree reaping (the remote branch *is* the registry, cf. id:a4e9/d190), and is the clean substrate for the 93fe one-writer drain — no security argument needed. Riku: then don't smuggle the security case in behind it; the containment *enforcement* rests on n=1 agent-error and is "observe before preventing" territory — decouple. Petra: ratify a **pilot**, not a fleet migration + OS-user sprint. Sage: worktree-per-session already ships; real-remotes is a drop-in swap of the integrator's merge *source* (`git fetch <copy>` vs merge local worktree branch), so relay-loop.js changes are modest.

**Copy mechanism (item 2).** Reflink-cp gives CoW independence (no hardlink aliasing) + warm trees; `/home` is btrfs (verified). Riku: reflink couples to btrfs + same-fs (fievel is Raspbian) → fallback is `git clone --local --no-hardlinks` (cold tree), documented not built; and reflink copies **dirty state** → gate on a clean *tracked* tree (keep untracked gitignored build dirs). Petra: no copy *abstraction* for one consumer (N=1); subvolume conversion out of scope.

**Push model (item 3).** The natural real-remotes model is one-writer: executor commits only to its own copy, integrator fetches + merges + is sole pusher. Riku: this changes the *contract* (executor stops pushing) but not *enforcement* (still runs as `tobias`, still holds the cred — credential removal is the deferred containment half); pick a pilot repo where dropping executor-push breaks nothing observable → **dotclaude-skills** (local meta-repo, we're already in it).

**Enforcement + 931c (item 4).** Riku: keep the deferred enforcement honestly observe-first — instrument the wrong-path-write class (via id:194e failure-ledger) so reopening is gated on a *measured rate*, not another anecdote. Archie: 931c splits into (a) build declarative model/effort defaults, (b) record the non-containment invariant (agent-type tool lists are NOT a security boundary — id:453a/5937). Sage: build (a) *after* the pilot (it may reshape the executor agent def); spec via structural/exec-smoke tests (sandbox-2ec4). Petra: do NOT reshape the in-flight 93fe children (id:5367/2062 on the worktree substrate) — real-remotes is an alternative substrate to converge with later, a follow-up, not a rewrite.

## Amendment session — Fable-5 closing pass (owner-requested at the closure gate)

Per the owner's standing preference (user-profile: adversarial second opinion is a gate on *recording*), a Fable-5 subagent gave an advisory closing pass on the ratified decisions (advisory only, never a gate; subagent-not-driver). It verified its claims this session (findmnt btrfs; dotclaude-skills has no manifest/build deps; claim/heartbeat state lives in `~/.config/relay` outside repos; contract v10 push posture). It surfaced **3 holes forcing amendments**, all owner-ratified:

- **H1 (amends D4a):** the instrument as first ratified counts only self-caught + integrator-caught writes, but a stray canonical write is **laundered by the mandatory git-diary-workflow auto-commit+push** into an anonymous user commit → never reaches the ledger. So the measured rate is a lower bound biased against the very quantity D1 gates on, and the deferral self-seals. Fix: dispatch/integrate `git status --porcelain`+HEAD diff → `wrong-path-write:SUSPECT` for unattributable deltas (honestly FP-labeled — no uid attribution exists); split category into self-caught / integrator-caught / suspect now; record caught-counts as a lower bound.
- **H2 (amends D2/D3):** `cp --reflink -r` copies `.git/config` verbatim → the executor copy keeps `remote.origin.url` + the credential; "never touches origin" is prose against a habit-error class. Fix: copy step runs `git -C <copy> remote set-url --push origin NO_PUSH` — zero-cost config hygiene, **not** the deferred uid-DAC containment (D1's defer-logic doesn't apply to a one-liner).
- **H3 (amends the success metric):** metric (ii) "warm-tree cheaper than cold clone **on a build-dep repo**" is unsatisfiable on dep-less dotclaude-skills → risk that "pilot passed" licenses fleet migration with the reflink economic premise untested. Fix: drop (ii) from this pilot (it's a *transport shakedown*, not safety evidence — the hazard dissolution is true by construction); gate fleet migration on a standalone warm-vs-cold timing on a real build-dep repo (zkm plugin / llm-from-scratch, measure-only).

Six **hardening** items folded into the action items (don't force reopening): copy under `git-lock-push.sh` flock + post `git fsck --connectivity-only`; use `cp -a --reflink=always` (preserve mtimes, else git rehashes the whole tree and eats the warm win); pin the copy destination under `/home` (reflink can't cross to tmpfs); handle untracked ride-alongs (`RELAY_STATUS.md`, `persona-events/`, `settings.local.json`, live lock files) via exclude-list or a contract line; **factual correction** — contract v10 *already* has commit-only executors + integrator-sole-pusher, so D3's real delta is **transport** (worktree branch → fetch-from-copy) + copy-analogues for rule 5b's clean-worktree gate and `worktree-retire.sh`, with a `contract vN` bump; pilot rounds stay supervised and don't interleave with drain runs on this repo.

Fable **defended** (do not reopen): the reflink-over-hardlink-clone choice (no CoW/gc gotcha at the git layer), D1's decoupling logic, the coordination-state non-issue (claim/heartbeat state is outside repos), D3's direction. This is the **second dogfood proof** that the manual closing pass catches ratifiable holes (4 last time, 3 now) — strengthens the case for building `id:7e87 --fabled`.

## Decisions

- **D1 — Decouple; pilot the topology now.** Real-remotes topology is decided on ergonomic/architectural merit; uid-scoping *containment* is gated on evidence. Adopt real-remotes as a **pilot on dotclaude-skills only**; defer fleet migration AND all uid-DAC enforcement until a measured failure rate justifies it. *Out of scope:* fleet migration, uid/DAC enforcement, LLM-tier OS users.
- **D2 — Copy mechanism = `cp -a --reflink=always`** on the plain dir, gated on a **clean tracked tree** (keep untracked gitignored build dirs for warmth), destination under `/home`, taken under the `git-lock-push.sh` flock with a post-copy `git fsck --connectivity-only`. Document `git clone --local --no-hardlinks` as the non-btrfs fallback but do **not** build it. *Out of scope:* subvolume conversion, a pluggable copy abstraction (N=1).
- **D3 — One-writer transport.** Piloted-repo executor commits to its own copy only and never touches origin; the copy's push URL is neutralized (`remote set-url --push origin NO_PUSH`, per H2); the integrator `git fetch`es from the copy path, merges `--no-ff`, and is sole pusher. This is a **transport** change to the executor contract (worktree branch → fetch-from-copy) — contract v10 already has commit-only executors — plus copy-analogues for rule 5b's clean-worktree gate and `worktree-retire.sh`; bump the `contract vN` marker. *Out of scope:* credential *removal* / ssh-push infra (id:38bf) — the deferred containment half.
- **D4a — Instrument now, gate enforcement on a measured rate (as amended by H1).** Add a `wrong-path-write` failure-ledger category (id:194e substrate) built as a dispatch/integrate `git status`+HEAD diff → self-caught / integrator-caught / **SUSPECT** split, SUSPECT honestly FP-labeled, caught-counts recorded as a lower bound. Reopen uid-DAC enforcement only on a measured rate (~5–10 pool runs) or immediately on the first unrecoverable-cost collision.
- **D4b — Record the non-containment invariant now; build 931c defaults after the pilot.** Bank the invariant "agent-type tool lists are NOT a security boundary — complementary ergonomics only" (generalizes id:453a). Build id:931c(a) declarative model/effort defaults **after** the pilot settles the executor agent def, spec'd via structural/exec-smoke tests. *Out of scope:* treating agent-types as containment.
- **Success metric (amended by H3):** the pilot is a **transport shakedown** — passes iff a dispatch+integrate round completes via copy→fetch→merge with the full suite green and no manual intervention. Fleet migration is separately gated on a standalone warm-vs-cold copy timing on a real build-dep repo.
- **Sequencing:** do not reshape the in-flight 93fe children (id:5367/2062, worktree substrate); real-remotes convergence with them is a follow-up.
