## Decisions
- **D1 — No auto-integration; park unmerged orphans on discovery.** On commits-ahead at discovery (`relay-loop.js:~565-569`): `git worktree remove --force <dir>` then `git branch -m relay/<runId>-<v> relay/orphan/<runId>-<v>` (canonical `relay/orphan/*` namespace; commit stays reachable). Emit ONE summary line, not a per-round handback — removing the dir is what stops the `ls worktrees/` scan re-surfacing it. *Out of scope:* auto-integration; the old per-round re-surfacing handback. <!-- id:689c -->
- **D2 — Scripted `/relay reconcile` mode (human-invoked only).** Enumerate `relay/orphan/*` across managed repos; per-branch {integrate: verify clean main → `git merge --no-ff` → `ckpt-tag.sh` → `git-lock-push.sh --ff-only` | discard: `git branch -D` | leave}. MUST reuse `ckpt-tag.sh` + `git-lock-push --ff-only` so a human can't skip the checkpoint tag or race the pool's push. Never auto-triggered. *Out of scope:* pool auto-running reconcile; CAS-plumbing merges (must be `--no-ff`). <!-- id:3313 -->
- **D3 — Suppress-redispatch ("don't repeat the expensive session in vain").** Discovery binds each `relay/orphan/*` → its ROADMAP item via `git show --stat`; if that item is still OPEN, suppress fresh dispatch + surface one line with a `relay-burn.sh --run <runId>` cost hint (best-effort). `/relay reconcile` lists orphans ordered by cost-hint, fallback commit-count/recency. Ambiguous binding → default to suppress. No new manifest — the refs ARE the registry. *Out of scope:* cost-recovery infra; a retry counter. <!-- id:1f53 -->
- **D4 — Pre-dispatch "don't start expensive items in vain": forward-flag only.** Cross-reference id:a4e9 ↔ id:4267 (quota-stop seatbelt) at the dispatch/quota-gate locus so "don't BEGIN an item you can't finish under remaining quota" is tracked there. *Out of scope of the build:* cost-prediction-before-dispatch (quota gate owns the locus). <!-- id:a692 -->
- **D5 — Existing-orphan disposal: moot.** The 9 are already reaped; collaib's A/B impl was discarded by a later run and is not recoverable here. No disposal action.
- **D6 — `git fsck` dangling-commit audit in `/relay review`: filtered ADVISORY follow-on.** Parking keeps the ref → a dangling commit only arises from a bug in D1/D2, so fsck audits that logic. Occasional: `git fsck --no-reflogs --unreachable --no-progress | grep commit`, FILTERED to relay-shaped commits (`git show --stat` touches `ROADMAP.md`/`RELAY_LOG.md`/`REVIEW_ME*`, or message `^(merge\(relay\)|handoff|review|todo|roadmap)`). ONE ADVISORY line, never auto-acts. Kill-if-noisy: remove if 0 real hits in ~20 review runs. Follow-on after D1–D3. <!-- id:122f -->
