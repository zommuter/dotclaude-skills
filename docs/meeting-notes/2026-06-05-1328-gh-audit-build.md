## Plan

- New sibling script `meeting/gh-audit.sh` (two subcommands: `open`, `search <query>`).
- Remote detection: scan all remotes for `github.com` URL (not just `origin` — in dotclaude-skills the GitHub remote is named `github`); extract `owner/repo`; fail-soft (exit 0, no stdout) if no GitHub remote, `gh` unavailable, or unauthenticated.
- Output: one `issue|pr #N  <title>  <url>` line per item; cap via `GH_AUDIT_LIMIT` env var (default 10) with suppression notice.
- Log: TSV line to `~/.claude/logs/meeting-gh-audit.log` per run.
- Wire into SKILL.md: `gh-audit.sh open` runs after orphan scans in both no-arg step 1 and subject step 2 (advisory label); `gh-audit.sh search` runs in setup step 5 for subject mode (GitHub prior art label, not advisory).
- Allowlist: added `gh-audit.sh` to `meeting_FILES/EXEC/ALLOW` in Makefile; `make install-allowlist` generated 8 entries.
- F2 multi-repo walk and id-correlation deferred (no consumers / folds into F3 write session).

## Decisions

- **`gh-audit.sh`** is the sole new artifact. `orphan-scan.sh` is left untouched (stays fast/offline).
- **Remote detection** scans all remotes, not just `origin`, to handle repos where the GitHub remote has a non-standard name.
- **Fail-soft** exit 0 / no stdout on any error — SKILL.md callers do not need error handling.
- **No id-correlation** in v1 (informational-read-only). Correlation folds into F3 when issue-writing is on the table.
- **F2 multi-repo walk** deferred; `find-todos.sh` remains the reuse primitive when built.
- *Out of scope:* mutating GitHub state, gh native search result scoring, F2, id-correlation.
